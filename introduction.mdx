## What is MinIO?

MinIO is a high-performance, S3-compatible object storage server released under the GNU AGPL v3.0 license. It is designed for speed and scalability, powering AI/ML, analytics, and data-intensive workloads with industry-leading performance.

MinIO implements the full Amazon S3 API, making it a drop-in replacement for S3-based applications. You can use any S3-compatible tool, SDK, or library with MinIO without modification.

## Architecture overview

MinIO is a single binary that acts as both the object storage server and the web-based console. In distributed mode, multiple MinIO instances form a cluster where data is striped using erasure coding across drives.

## Deployment modes

<Tabs>
  <Tab title="Single node">
    Run MinIO on a single machine with one or more drives. Suitable for development, testing, and small workloads.

    ```bash
    minio server /data
    ```
  </Tab>
  <Tab title="Single node, multiple drives">
    Run MinIO on a single machine with multiple drives for erasure coding protection.

    ```bash
    minio server /data{1...4}
    ```
  </Tab>
  <Tab title="Distributed (multi-node)">
    Run MinIO across multiple nodes, each with multiple drives. Provides the highest availability.

    ```bash
    minio server https://minio{1...4}.example.com/data{1...4}
    ```
  </Tab>
</Tabs>
