## Defense in depth

MinIO's security model addresses four distinct concerns:

| Layer | Mechanism | Purpose |
|---|---|---|
| Transport | TLS/SSL | Encrypts data in transit between clients and the server |
| Storage | SSE-S3, SSE-KMS, SSE-C | Encrypts objects at rest using a key hierarchy |
| Identity | IAM users, LDAP, OIDC | Authenticates who is making a request |
| Access | IAM policies, STS | Controls what authenticated identities are permitted to do |

## Cryptographic primitives

MinIO uses industry-standard primitives for all encryption operations:

- **PRF**: HMAC-SHA-256 for key derivation
- **AEAD cipher**: AES-256-GCM on CPUs with AES-NI; ChaCha20-Poly1305 on all others
- **Secret key size**: 256 bits for all internally generated keys
- **Secure channel chunk size**: 65,536 bytes; supports plaintexts up to 256 TiB

## Key hierarchy for server-side encryption

MinIO never uses an external key (EK) or KMS master key directly to encrypt object content. Instead, it builds a three-level key hierarchy:

1. **CMK** — The master key held in the KMS.
2. **EK (External Key)** — A data key generated by the KMS per object (SSE-S3) or provided by the client (SSE-C).
3. **KEK (Key Encryption Key)** — Derived from the EK, a random IV, and context values (bucket/object name) using HMAC-SHA-256. Never stored.
4. **OEK (Object Encryption Key)** — A randomly generated key that encrypts the object content. Stored encrypted in object metadata.

This design means that compromising any single layer does not immediately expose object content.

## Secure erasure

Because SSE-S3 encrypted objects require the KMS to decrypt, you can make all encrypted objects unreadable without deleting them by:

- **Sealing the KMS** — locks all SSE-S3 objects protected by its master keys.
- **Sealing or deleting a master key** — locks or permanently destroys all objects encrypted under that key.

## Authentication types

MinIO recognizes the following authentication types on incoming requests:

| Auth type | Description |
|---|---|
| `authTypeSigned` | AWS Signature V4 (standard API requests) |
| `authTypePresigned` | Pre-signed URL with V4 signature |
| `authTypeSignedV2` / `authTypePresignedV2` | Legacy AWS Signature V2 |
| `authTypePostPolicy` | S3 POST policy upload |
| `authTypeStreamingSigned` | Chunked streaming upload |
| `authTypeJWT` | JWT token (Console and admin API) |
| `authTypeSTS` | Temporary STS credentials |
| `authTypeAnonymous` | Unauthenticated (public bucket access) |
