---
title: "Object compression"
description: "Reduce storage costs by compressing objects in MinIO"
---

MinIO supports streaming compression for objects. Compression happens in-flight: objects are compressed before being written to disk. Reading a compressed object decompresses it on the fly — clients always see the original data.

MinIO uses the [`klauspost/compress/s2`](https://github.com/klauspost/compress/tree/master/s2) algorithm (an S2/Snappy variant). Key throughput characteristics:

- Write throughput: at least **500 MB/s per CPU core**, scaling with core count
- Read throughput: at least **1 GB/s**

On systems where raw disk I/O is below these thresholds — such as spinning disk arrays — enabling compression can increase overall throughput while simultaneously reducing storage usage.

## Enabling compression

Compression is disabled by default. Enable it with `mc admin config set` or environment variables.

<Tabs>
  <Tab title="mc admin config">
    ```bash
    # Enable compression with default extensions and MIME types
    mc admin config set myminio compression enable="on"

    # View current configuration
    mc admin config get myminio compression
    ```

    ```text
    compression extensions=".txt,.log,.csv,.json,.tar,.xml,.bin" mime_types="text/*,application/json,application/xml"
    ```
  </Tab>
  <Tab title="Environment variables">
    ```bash
    export MINIO_COMPRESSION_ENABLE=on
    export MINIO_COMPRESSION_EXTENSIONS=".txt,.log,.csv,.json,.tar,.xml,.bin"
    export MINIO_COMPRESSION_MIME_TYPES="text/*,application/json,application/xml"
    ```

    Environment variables override any `mc admin config` settings.
  </Tab>
</Tabs>

## Configuring extensions and MIME types

Compression applies to objects whose file extension **or** content type matches the configured lists. You can configure these independently.

```bash
# Compress only PDF files
mc admin config set myminio compression \
  extensions=".pdf" \
  mime_types="application/pdf"
```

```bash
# Compress all content regardless of extension or type
# (except the default excluded types — see below)
mc admin config set myminio compression \
  enable="on" \
  extensions="" \
  mime_types=""
```

<Note>
When using environment variables, set `MINIO_COMPRESSION_EXTENSIONS="*"` and `MINIO_COMPRESSION_MIME_TYPES="*"` to compress all content. An empty string in an environment variable does not have the same effect as it does in the `mc admin config` approach.
</Note>

## Content excluded from compression

Certain file types are always excluded, even when "compress everything" is configured. These formats are already compressed and would not benefit — or could even increase in size.

<AccordionGroup>
  <Accordion title="Excluded file extensions">
    | Extension | Format |
    | :--- | :--- |
    | `.gz` | GZIP |
    | `.bz2` | BZIP2 |
    | `.rar` | WinRAR |
    | `.zip` | ZIP |
    | `.7z` | 7-Zip |
    | `.xz` | LZMA |
    | `.mp4` | MP4 video |
    | `.mkv` | MKV media |
    | `.mov` | MOV video |
  </Accordion>
  <Accordion title="Excluded MIME types">
    | Content-Type |
    | :--- |
    | `video/*` |
    | `audio/*` |
    | `application/zip` |
    | `application/x-gzip` |
    | `application/x-bz2` |
    | `application/x-compress` |
    | `application/x-xz` |
  </Accordion>
</AccordionGroup>

Attempting to re-compress already-compressed data has a minimal performance impact. The s2 algorithm detects incompressible input and processes it at over 2 GB/s per core, so excluding known binary formats manually is not required for performance — only storage efficiency.

## Compression with encryption

<Warning>
Combining compression with encryption is not safe in all configurations. If an attacker can influence plaintext content and observe ciphertext size changes, they may be able to infer information about the plaintext (see [CRIME TLS](https://en.wikipedia.org/wiki/CRIME) for background). Consult your security team before enabling this combination.
</Warning>

By default, MinIO disables compression when SSE is active. To explicitly enable both:

<Tabs>
  <Tab title="mc admin config">
    ```bash
    mc admin config set myminio compression allow_encryption=on
    ```
  </Tab>
  <Tab title="Environment variable">
    ```bash
    export MINIO_COMPRESSION_ALLOW_ENCRYPTION=on
    ```
  </Tab>
</Tabs>

## Verifying compression

Upload an object and compare its on-disk size to confirm compression is active:

```bash
# Upload a compressible text file
mc cp access.log myminio/logs/

# List the data directory to see the compressed size on disk
mc ls --versions myminio/logs/
```

Alternatively, inspect the raw data directory on the MinIO host — compressed objects occupy less space than their original size.

## Performance considerations

- Compression is CPU-bound. On servers with many cores the throughput scales linearly.
- Objects that are already compressed (images, videos, archives) pass through the compressor at very high speed without meaningful size reduction. Excluding known incompressible types is a best practice for throughput efficiency.
- Decompression speed exceeds 1 GB/s per core, so read performance is rarely a bottleneck.
- Spinning disk environments typically see a net throughput improvement because the reduced I/O outweighs the CPU overhead.
