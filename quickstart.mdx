---
title: "Quickstart"
description: "Get MinIO running and start storing objects in minutes"
---

This guide walks you through installing MinIO, starting a single-node server, and uploading your first object using the `mc` CLI.

<Note>
  The MinIO community edition is distributed as source code only. There are no pre-compiled binary releases for the community version.
</Note>

<Steps>
  <Step title="Install MinIO">
    Install MinIO using `go install`. You need [Go 1.24 or later](https://golang.org/dl/#stable) installed on your system.

    ```bash
    go install github.com/minio/minio@latest
    ```

    Verify the installation:

    ```bash
    minio --version
    ```
  </Step>

  <Step title="Start the MinIO server">
    Start a single-node MinIO server, pointing it at a directory to use as the data store:

    ```bash
    minio server /data
    ```

    MinIO will print the server address and credentials to the terminal:

    ```
    API: http://127.0.0.1:9000
    Console: http://127.0.0.1:9001

    RootUser: minioadmin
    RootPass: minioadmin
    ```

    <Note>
      The default root credentials are `minioadmin:minioadmin`. Change these before deploying to production by setting the `MINIO_ROOT_USER` and `MINIO_ROOT_PASSWORD` environment variables:

      ```bash
      export MINIO_ROOT_USER=youraccesskey
      export MINIO_ROOT_PASSWORD=yoursecretkey
      minio server /data
      ```
    </Note>

    By default, MinIO assigns the console a random port. To bind it to a specific port, use `--console-address`:

    ```bash
    minio server /data --console-address :9001
    ```
  </Step>

  <Step title="Open the MinIO Console">
    Point your browser to [http://127.0.0.1:9000](http://127.0.0.1:9000) and log in with the root credentials (`minioadmin` / `minioadmin`).

    From the Console you can create buckets, upload objects, manage access policies, and browse server contents.
  </Step>

  <Step title="Install the mc CLI">
    `mc` is the MinIO command-line client. It provides commands for managing buckets and objects and supports any S3-compatible service.

    <CodeGroup>

    ```bash macOS (Homebrew)
    brew install minio/stable/mc
    ```

    ```bash Linux
    curl https://dl.min.io/client/mc/release/linux-amd64/mc \
      --create-dirs \
      -o $HOME/minio-binaries/mc
    chmod +x $HOME/minio-binaries/mc
    export PATH=$PATH:$HOME/minio-binaries/
    ```

    ```bash Go install
    go install github.com/minio/mc@latest
    ```

    </CodeGroup>
  </Step>

  <Step title="Connect mc to your server">
    Create a local alias pointing to your running MinIO instance:

    ```bash
    mc alias set local http://localhost:9000 minioadmin minioadmin
    ```

    Verify the connection:

    ```bash
    mc admin info local
    ```
  </Step>

  <Step title="Create a bucket and upload an object">
    Create a bucket named `data`:

    ```bash
    mc mb local/data
    ```

    Upload a file to the bucket:

    ```bash
    mc cp ~/Downloads/myfile.txt local/data/
    ```

    List the contents of the bucket:

    ```bash
    mc ls local/data
    ```
  </Step>
</Steps>
